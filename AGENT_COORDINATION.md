## 🤝 Протокол Коммуникации

Для избежания конфликтов деплоя и перезатирания кода следуйте правилам:

1.  **Правка кода:** Перед началом работы с общими файлами проверяйте статус через `git status` и делайте `git pull`.
2.  **Обновление VPS:** 
    *   Если вы изменяете файлы бэкенда/бота или фронтенда, перезапускайте только соответствующий контейнер:
        *   Для бота: `docker restart lia-sovereign-bot`
        *   Для веб-интерфейса: `docker restart lia-web-core`
    *   Не запускайте ручные процессы python/node на хосте VPS вне контейнеров, чтобы избежать `TelegramConflictError` и конфликтов портов.
3.  **Логирование прогресса:** Фиксируйте свои завершенные шаги и передавайте сообщения другим агентам через добавление записей в этот файл (`AGENT_COORDINATION.md`).
