### Как обновить и заверить хронологию

Полную ленту коммитов с метками времени можно выгрузить командой:

```powershell
git log --reverse --pretty=format:"%H | %ad | %an <%ae> | %s" --date=iso > AUTHORSHIP_gitlog.txt
```

Для усиления доказательности рекомендуется:

1. **Подписать коммиты GPG-ключом** (`git commit -S`) — подтверждает личность автора.
2. **Запушить в публичный/приватный remote** (GitHub) — внешняя независимая метка времени.
3. **Создать аннотированный git-tag релиза** с датой:
   ```powershell
   git tag -a v6.0-authorship -m "Authorship snapshot 2026-06-22" c0a9f69
   ```
4. Сделать **timestamping** контрольной суммы репозитория через доверенный TSA
   (RFC 3161) или OpenTimestamps (привязка к блокчейну).

### Как пересчитать / добавить артефакты

```powershell
certutil -hashfile "Synapse_Manifesto.pdf" SHA256
certutil -hashfile "lia_bot_official_logo_1777671969621.png" SHA256
```

## 4. Процедура депонирования (фиксация даты у третьей стороны)

Чтобы дата стала юридически значимой, привяжите её к независимому источнику:

1. **OpenTimestamps / блокчейн-метка** — бесплатно, привязывает SHA-256 к
   времени блока биткоина (неоспоримая метка времени).
2. **RFC 3161 Timestamp Authority** — квалифицированная электронная метка времени.
3. **Депонирование в авторском обществе / у нотариуса** — официальная регистрация
   экземпляра произведения с датой.
4. **Письмо самому себе / в архив** — отправить запечатанный архив с хешами на
   собственный e-mail (серверная метка времени) или в облако с историей версий.
5. **Публикация** манифеста и спецификации в датируемом источнике
   (репозиторий, блог-платформа с датой публикации).

> Минимальный практичный набор: GPG-подписанный git-tag + push в GitHub +
> OpenTimestamps на SHA-256 манифеста.
