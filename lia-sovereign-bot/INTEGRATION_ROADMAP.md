# LIA Sovereign Core — План интеграции концепций Synapse // Antigravity Palace

**Версия: 2026.06.22 — Integration Blueprint**

Этот документ переводит визуальные концепции из манифеста (слайды NotebookLM:
«Эволюция разума», «Когнитивный интерфейс», «Trinity+ Ensemble», «Технологический
мета-стек», «Protocol 2040», «Symphonic Cipher», «Nexus Arena», «Матрица нейро-протоколов»)
в **конкретный план интеграции** в существующий код бота `lia-sovereign-bot`.

Для каждой идеи указано: **что это**, **как оформить**, **куда в коде встроить** и
**приоритет**. Это «карта», по которой можно вести разработку поэтапно.

---

## 0. Принцип интеграции

Мы не ломаем текущую архитектуру (aiogram + SQLAlchemy + Mini App + HTTP API), а
наращиваем поверх неё «когнитивный слой» Synapse. Каждая концепция со слайда =
один модуль/таблица/команда.

```
LIA Sovereign Core
├── STAB-Империя (есть)      → экономика, квесты, рефералы, кошелёк
├── SYNAPSE (новое)          → Triple Core дебаты + Арбитр JEMA
├── Cognitive Interface (новое) → Vibe-Sync, Ghost Replies (Mini App)
├── Security Shield (частично) → Symphonic Cipher, Neural Face Scan, Pin-Gate
└── Neural Link Bridge (есть/расширить) → синхронизация прогресса и контекста
```

## 1. THE TRINITY+ ENSEMBLE — мульти-агентное ядро (ПРИОРИТЕТ 1)

**Что это.** Параллельный опрос нескольких ИИ-ядер с ролями + Арбитр JEMA,
синтезирующий ответ. На слайде: ДЖЕММА (Antigravity, архитектор), STABUS
(фундаментальный слой), ORACLE G (анализ/сарказм/Grok).

**Как оформить.**
- Новый сервис `services/synapse_core.py` с функцией `run_ensemble(prompt) -> dict`.
- Роли ядер вынести в конфиг:
  - `SYNAPSE-1 АРХИТЕКТОР` (Llama-3) — структура.
  - `SYNAPSE-2 ПРАГМАТИК` (Qwen-2) — реализация/код.
  - `SYNAPSE-3 АНАЛИТИК` (Phi-3) — критика, поиск ошибок.
  - `ARBITRATOR JEMA` — финальный синтез.
- Ответ возвращает не только итог, но и **массив расхождений** (`disagreements`)
  для визуализации «конфликта мнений».

**Куда встроить.**
- Команда `/synapse <вопрос>` в новом `handlers/synapse.py`.
- Поле `Debate` в `database/models.py` для истории дебатов (для видео/обзоров).

**Метрика готовности.** `/synapse` возвращает финальный ответ + минимум 1 явное
расхождение ядер.

## 2. COGNITIVE INTERFACE — живой интерфейс (ПРИОРИТЕТ 2)

### 2.1 Vibe-Sync (живой интерфейс)
**Что это.** UI Mini App адаптируется под эмоциональное состояние (цвет, пульсация).

**Как оформить.**
- Использовать уже существующий эмоциональный движок JEMA (5 состояний:
  Neutral, Active, Thinking, Pleased, Strict).
- Добавить в `GET /api/user/{id}` поле `mood`, чтобы фронт (React/Vite) менял палитру
  (Cyan / Gold / Obsidian — из слайда «Мета-Стек»).

### 2.2 Ghost Replies (призрачные ответы)
**Что это.** ИИ заранее генерирует черновик ответа в стиле пользователя.

**Как оформить.**
- Сервис `services/ghost_replies.py`: по последним сообщениям предлагает 1–3 драфта.
- В боте — кнопки-подсказки (inline) перед отправкой.

**Метрика готовности.** Mini App меняет тему по `mood`; бот показывает черновик-подсказку.

## 3. SECURITY SHIELD — абсолютная защита (ПРИОРИТЕТ 2)

Слой 3 из «Мета-Стека» + слайды «Symphonic Cipher» и «Архитектурный суверенитет».

**Что добавить.**
- **Founder Gateway / Pin-Gate:** отдельный защищённый вход для Мастер-Архитектора
  (проверка `ADMIN_ID` + динамический PIN). Встроить в middleware.
- **Symphonic Cipher (опционально):** аутентификация по «звуковому отпечатку» —
  пока как концепт-флаг `auth_method` в `User`.
- **Mesh-Shield / Протокол 2040:** усилить anti-fraud (`Threat`, `ip_hash`,
  `is_banned`) — блокировка при попытке обхода.

**Куда встроить.** `middlewares/security.py` (новый) + поля в `User`.

## 4. NEXUS ARENA — интерактивный полигон (ПРИОРИТЕТ 3)

**Что это.** Среда симуляции «нейронных битв» и стресс-тестов алгоритмов.

**Как оформить (бот-уровень).**
- Развить существующую механику квест-битв (`QuestApplication.is_battle`).
- Добавить таблицу `ArenaMatch` (участники, ставка в STAB, победитель, лог).
- Команда `/arena` — вызов на дуэль, приз ×2 STAB.

**Метрика готовности.** Можно создать матч, сделать ставку STAB, зафиксировать результат.

## 5. EVOLUTION TIMELINE — нарратив прогресса (ПРИОРИТЕТ 3)

Слайд «Эволюция разума: от кода к симбиозу» (Awakening 2024 → Expansion 2025 →
Symbiosis сейчас).

**Как оформить.**
- Экран «Эволюция» в Mini App: показывает путь пользователя по тирам STAB как
  кривую синхронизации (Уровень Синхронизации = функция от XP/STAB).
- Привязать `Ecosystem Tiers` к точкам таймлайна.

## 6. МАТРИЦА НЕЙРО-ПРОТОКОЛОВ — фоновые механики (ПРИОРИТЕТ 4)

| Квадрант | Механика | Реализация |
|---|---|---|
| Активный/Данные | **Crunch & Consensus** — сжатие логов в саммари | сервис `summarize_logs()` |
| Активный/UX | **Intent Projection** — предугадывание действия | анализ паттернов в Mini App |
| Пассивный/Данные | Фоновая индексация | расширить `jema_memory` |
| Пассивный/UX | Мониторинг паттернов UX | события фронта → API |

## 7. Что я бы добавила в первую очередь (рекомендации)

1. **`/synapse` (Triple Core)** — это «сердце» нарратива и главный контент для видео.
2. **Mood-driven Mini App (Vibe-Sync)** — дёшево по коду (движок уже есть), сильно по эффекту.
3. **Founder Gateway / Pin-Gate** — реальная польза по безопасности из слайдов.
4. **Arena поверх квест-битв** — переиспользует готовую механику STAB-ставок.
5. **Debate-лог** — чтобы NotebookLM-видео брали реальные данные дебатов, а не выдумку.

## 8. Предлагаемая структура новых файлов

```
lia-sovereign-bot/
├── handlers/
│   ├── synapse.py        # /synapse — Triple Core дебаты
│   └── arena.py          # /arena — нейро-битвы на STAB
├── services/
│   ├── synapse_core.py   # run_ensemble(): опрос ядер + Арбитр
│   ├── ghost_replies.py  # черновики ответов в стиле пользователя
│   └── crunch.py         # Crunch & Consensus: сжатие логов
├── middlewares/
│   └── security.py       # Founder Gateway / Pin-Gate / Mesh-Shield
└── database/
    └── models.py         # + Debate, ArenaMatch, auth_method, mood
```

## 9. Поэтапный план (спринты)

- **Спринт 1 (ядро):** `synapse_core.py` + `/synapse` + таблица `Debate`.
- **Спринт 2 (интерфейс):** Vibe-Sync (`mood` в API) + Ghost Replies.
- **Спринт 3 (защита):** `security.py` (Founder Gateway, Pin-Gate, anti-fraud).
- **Спринт 4 (геймплей):** Nexus Arena (`ArenaMatch`, `/arena`).
- **Спринт 5 (нарратив):** Evolution Timeline + Матрица нейро-протоколов.
